# Статус: tzhk-cms + сайты

Один мульти-тенант Payload CMS (`tzhk-cms`) кормит несколько декуплд-сайтов.
Каждый сайт читает контент по REST (ISR) и **всегда имеет локальный fallback**:
с пустым `CMS_URL` рендерится встроенный контент.

```
сайты (Next.js)  ──►  @/lib/content (адаптеры → наши типы)  ──►  Payload REST (tzhk-cms)
                       fallback на seed, если CMS нет
```

Тенанты: `tatushka` (татушки), `alfaag` (мамин сайт, вязание), `portfolio` (моё портфолио).

_Обновляется в каждом PR — чтобы не пересобирать статус с нуля._

## Деплой (после мёржа PR)
1. Деплой `tzhk-cms` → миграции применятся сами.
2. Контент: кнопка «Import content» в админке (или `seed:content`) для `tatushka` и `alfaag`.
3. Логин маме: Users → Create (tenant `alfaag`, не super-admin).
4. Env на фронтах: `CMS_URL`, `PAYLOAD_URL`, `CMS_TENANT_SLUG` → редеплой.
5. Заполнить реальные контакты/FAQ/отзывы в админке.
